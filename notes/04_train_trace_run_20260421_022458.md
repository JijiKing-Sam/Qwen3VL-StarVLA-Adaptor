# Qwen3-VL-OFT LIBERO Training Trace Run 20260421_022458

这份笔记对应一次真实训练链路 smoke run，用来学习 StarVLA 的训练数据流。

## 运行结论

本次不是纯 mock。它实际完成了：

```text
LIBERO LeRobot 数据读取
  -> DataLoader batch
  -> Qwen3-VL multimodal tokenization
  -> Qwen3-VL forward
  -> action token hidden state 抽取
  -> action head 预测动作块
  -> 和真实 target action 算 L1 loss
  -> backward
  -> gradient clip
  -> optimizer step
  -> scheduler step
```

为了在单张 4080 SUPER 上稳定跑通，只做了这些限制：

```text
max_train_steps: 2
per_device_batch_size: 1
freeze_modules: qwen_vl_interface
skip_final_save: true
WANDB_MODE: disabled
```

也就是说，这次主要训练 action head；Qwen3-VL backbone 参与 forward，但被冻结，不更新参数。最后不保存大模型 checkpoint，是为了避免 smoke run 额外写出数 GB 文件。

## 关键观察

第一步样本：

```text
instruction: put the bowl on the plate
image: 2 张 RGB PIL 图，224 x 224
target action: float16, shape [8, 7]
batch size: 1
Qwen last_hidden: bfloat16, shape [1, 199, 2560]
action_queries: bfloat16, shape [1, 8, 2560]
pred_actions: float32, shape [1, 8, 7]
loss: 0.026219910010695457
```

第二步样本：

```text
instruction: put the wine bottle on the rack
image: 2 张 RGB PIL 图，224 x 224
target action: float16, shape [8, 7]
Qwen last_hidden: bfloat16, shape [1, 200, 2560]
pred_actions: float32, shape [1, 8, 7]
loss: 0.025770975276827812
```

为什么 Qwen hidden length 第一步是 199、第二步是 200：

```text
语言文本不同，token 数可能不同；
图像 token + 文本 token + action token 拼在一起后，总序列长度自然可能变化。
```

为什么 target action 是 `[8, 7]`：

```text
8: action chunk horizon，本次一次预测未来 8 个动作
7: robot action dimension，通常是末端位姿/增量相关维度 + gripper 维度
```

为什么 `last_hidden.requires_grad=false`：

```text
本次 smoke run 显式 freeze 了 qwen_vl_interface。
Qwen3-VL backbone 仍然做 forward 产出 hidden states，但它不反传更新。
后面的 action head 输出 pred_actions，requires_grad=true，所以 action head 仍然参与训练。
```

